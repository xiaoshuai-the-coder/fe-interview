# Java 相关

这部分主要覆盖后端面试高频知识点，按「基础」与「工程实践」两部分组织。

## 基础

Java 基础部分需要掌握以下核心内容：

**集合框架**是必问内容。`ArrayList` 基于数组实现，查找快但增删慢；`LinkedList` 基于双向链表，增删快但查找需遍历。`HashMap` 底层是数组 + 链表/红黑树，需要理解扩容机制和哈希冲突处理；`ConcurrentHashMap` 通过分段锁或 CAS 实现线程安全。

**并发编程**围绕线程安全展开。需要理解 `synchronized` 与 `ReentrantLock` 的区别与适用场景；CAS 原理与 ABA 问题；线程池的核心参数（核心线程数、最大线程数、队列策略）与拒绝策略选择。

**JVM** 需要掌握内存模型（堆、栈、方法区等）、垃圾回收算法（标记清除、复制、标记整理）以及常见垃圾回收器（Serial、Parallel、CMS、G1）的特点。遇到 OOM 时要会分析堆转储文件。

**Spring 核心**也是面试重点。IoC（控制反转）通过 BeanFactory/ApplicationContext 管理对象依赖；AOP（面向切面编程）基于代理模式实现，常用于日志、事务、权限等横切关注点。

## 工程与实践

**性能优化与稳定性保障**是工程能力的体现。慢查询定位需要会用 EXPLAIN 分析执行计划；接口性能优化可从缓存、异步、分页等角度入手；限流、熔断、降级是保障系统可用性的常用手段。

**线上问题排查**需要熟悉常用命令：`jstack` 查看线程状态、`jmap` 分析内存分布、`jstat` 监控 GC 情况。结合日志与监控从现象定位根因是排查问题的基本思路。
