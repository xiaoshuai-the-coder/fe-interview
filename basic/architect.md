# 架构设计相关

## 项目整体性能优化（回答框架）

面试里谈“项目性能优化”，最好从**定位问题 → 制定策略 → 验证收益 → 防止回退**的闭环来讲，而不是只罗列技巧。

### 1) 先定位：性能瓶颈在哪里

- **指标体系**：以用户体验为核心，常见指标包括 FCP/LCP/CLS/TTI、交互延迟（INP）、错误率与白屏率等。
- **采集方式**：线上 RUM（真实用户监控）+ 离线压测/本地 Profile（Performance/CPU/Memory）。
- **分层定位**：区分是网络、渲染、JavaScript 执行、内存/GC、还是后端接口慢导致的等待。

### 2) 再分治：按层做优化

- **加载性能**：减少资源体积（压缩、tree-shaking）、按需加载（code splitting）、缓存策略（强缓存/协商缓存）、CDN、图片优化（格式/尺寸/懒加载）。
- **运行时性能**：减少不必要的渲染与计算（memo、虚拟列表、事件节流/防抖）、避免长任务（切片、requestIdleCallback）、合理拆分组件与状态。
- **渲染与布局**：减少 layout thrashing，避免强制同步布局；动画尽量用 transform/opacity；合理使用骨架屏与占位。
- **稳定性与容错**：超时与重试、降级与兜底、错误边界与监控告警，确保优化不以稳定性为代价。

### 3) 最后验证：用数据说话

- **对比方法**：优化前后同一场景、同一网络与设备条件下对比，避免“体感优化”。
- **回归机制**：CI 中加入体积阈值、关键链路性能检测、监控面板告警，防止后续迭代性能回退。
