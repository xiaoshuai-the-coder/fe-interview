# 跨端方案

## Electron

### 多进程架构

Electron 采用主进程 + 渲染进程的双进程架构：
- **主进程**：基于 Node.js，负责应用生命周期管理、窗口创建、系统原生 API 调用
- **渲染进程**：基于 Chromium，运行 Web 页面，隔离于主进程

### 通信机制

主进程与渲染进程通过 IPC 通信：
- `ipcMain` 在主进程监听，`ipcRenderer` 在渲染进程发送
- 采用 `invoke/handler` 模式，基于 Promise 实现，异步非阻塞

### 安全机制

- **Context Isolation**：开启后隔离上下文，防止渲染进程直接访问 Node.js
- **Preload 脚本**：`contextBridge` 暴露受限 API，避免暴露全部 Node API
- **沙箱模式**：启用后渲染进程完全浏览器化，无法访问系统资源

## 小程序

### 渲染架构

小程序采用双线程模型：
- **逻辑线程**：运行 JavaScript，负责业务逻辑、数据处理
- **渲染线程**：运行 WXML/WXSS，负责页面渲染

### 通信方式

两层线程通过 JSBridge 通信：
- 逻辑层将数据传递给渲染层
- 渲染层触发用户交互事件传回逻辑层
- 通信过程有序列化开销，需注意数据传输体积
