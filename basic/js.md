# JavaScript 相关知识点

## 寄生组合继承

寄生组合继承常被认为是 ES5 时代**最接近 ES6 `class extends`** 的继承实现方式之一，核心目标是同时解决两类问题：**避免多次调用父类构造函数**、并且**让实例与原型链都正确继承**。

### 为什么需要它

在一些常见继承写法里，容易遇到下面的问题：

- **原型链继承**：子类实例能继承父类原型方法，但**无法向父类构造函数传参**，并且多个实例可能共享引用类型属性导致互相影响。
- **构造函数继承（借用构造函数）**：可以传参，实例属性独立，但**无法继承父类原型方法**。
- **组合继承**：既能传参又能继承原型方法，但会出现**父类构造函数被调用两次**（一次用于设置子类原型，一次用于创建子类实例），产生不必要的开销与冗余属性。

### 它的优点

寄生组合继承通过“**只在实例化时调用一次父类构造函数** + **子类原型通过 `Object.create` 继承父类原型**”来达成：

- 可以在子类构造函数里用 `Parent.call(this, ...)` **向父类传参**。
- 子类实例之间的属性互不影响（引用类型也不会共享）。
- 子类能继承父类原型上的方法，同时**不会额外多调用一次父类构造函数**。
- 这也是 ES6 `class` 语法糖背后（概念上）要达到的效果：**实例属性来自构造函数，方法来自原型链**。
